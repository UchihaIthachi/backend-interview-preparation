# Basic Questions

## When to use Node.js?

If your application is: 
1. I/O intensive 
2. Less CPU intensive computation  
3. Team has expertise in JS and no significant requirements of backend. 
4. Have knowledge of JS and want to do backend engineering, Nodejs is a good option to start with. 
5. Less verbose syntax and quicker to iterate. 

Node.js will work significantly well for these usecases. You can do child threading and processing to perform intensive tasks. But using Node.js for other usecase might impact the performance.
