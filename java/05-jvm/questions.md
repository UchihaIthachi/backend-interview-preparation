# JVM — Interview Question Bank

1. What happens after `javac` compiles a `.java` file — walk through class loading?
2. What is the class-loading lifecycle (loading, linking, initialization)?
3. Heap vs stack vs metaspace?
4. Interpreter vs JIT compiler?
5. Minor vs major/full collection?
6. What causes `StackOverflowError`?
7. What causes `OutOfMemoryError`?
8. How would you investigate a memory leak?
9. How would you investigate high CPU usage in a running JVM?
10. What information does a thread dump provide, and how do you capture one?
