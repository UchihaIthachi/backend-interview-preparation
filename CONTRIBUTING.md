# Contributing (to yourself, future you)

This is a personal study repo, but treat it like a real one:

- One topic per commit. Don't batch five unrelated notes into one commit.
- Use conventional commit prefixes:
  - `docs(java): ...` — notes
  - `feat(dsa): ...` — new solved problem
  - `feat(go): ...` — new project code
  - `test(java): ...` — added tests
  - `docs(system-design): ...` — new design doc
- Keep files under ~400 lines. Split into a new file once a topic outgrows that.
- Every topic file should be revision-ready: assume you'll re-read it at 11pm before an interview.
- Don't create empty placeholder files "for structure." Create a file when you have
  something real to put in it.
