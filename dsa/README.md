## How to practice

1. Read the pattern in `patterns.md` before looking at any specific problem.
2. Attempt the problem cold for 15-20 minutes.
3. If stuck, read the brute-force approach only, not the optimized one.
4. Implement the optimized approach yourself, then compare to the reference solution.
5. Explain your solution out loud in under 2 minutes — this is what the actual
   interview will require, not just working code.
