# Daily Study Routine

## Weekday (170 minutes total)

```text
45 min   Concept review (Java or Go, alternate days)
45 min   Coding or debugging exercise
30 min   Interview questions — answer out loud, not just in your head
30 min   Project improvement (add one feature, one test, one fix)
20 min   Convert today's notes into a topic file / blog draft
```

## Weekend

```text
Saturday
  -> Build one complete feature end-to-end
  -> Solve one system-design problem fully
  -> Run one mock interview (self-recorded or with a peer)

Sunday
  -> Review the week's mistakes
  -> Update cheat sheets
  -> Plan next week's 5 DSA problems by pattern
  -> Update roadmap/progress-tracker.md
```

## Rule of thumb

If you can't explain a topic out loud in under 90 seconds without notes,
it's not ready for an interview yet — go back to the topic file and rewrite
the "Short Interview Answer" section until it is.
